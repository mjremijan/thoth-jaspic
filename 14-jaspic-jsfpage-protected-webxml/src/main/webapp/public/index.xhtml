Can I protect a JSF page with a web.xml <security-constraint>, @DeclareRoles, and with identity-management groups in glassfish-web.xml?

Yes!

This page is /public/index.xhtml and there are no security settings in web.xml for this directory. So you can see this page fine.

The link below is to /executive/index.xhtml. The /executive directory is protected by the ExecutiveRole role. So, if you click it, you'll get the standard HTTP Status 403 - Forbidden page.

The link below is to /customersupport/index.xhtml. The /customersupport directory is protected by the CustomerSupportRole role. You have this role, so you should be able to access it with no problems.
