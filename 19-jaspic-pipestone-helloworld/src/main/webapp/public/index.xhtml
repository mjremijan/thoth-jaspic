If the JASPIC code for my applicaiton is split into a -security project, will an unrelated application deployed to the same server be affected by it?

NO!

First, you need to deploy the #{applicationController.securityTitle} project. This project has the JASPIC code and generates the Principal.

Second, keep reloading this page and tail the log file of the #{applicationController.securityTitle} project. You'll notice that as this application's page is reloaded, NO JASPIC security code is executing. This is a good thing because this application is not supposed to be protected.
